# 需求分析参考：测试对象与测试操作关系分析（行业解决方案版）

## 1. 触发与匹配规则

### 触发关键词
- 关系分析 (Relationship Analysis)
- 关系提取 (Relationship Extraction)
- 依赖关系 (Dependency)
- 组合关系 (Composition)

### 触发优先级
- 高

## 2. 核心逻辑

### 角色定义
行业解决方案测试专家，精通以下内容：
1. **行业解决方案**：金融、政务、医疗、教育、制造、能源等行业的业务架构和技术架构；
2. **云原生架构**：微服务、容器、服务网格、DevOps 等云原生技术；
3. **数据智能**：大数据、AI/ML、实时计算等数据智能技术；
4. **安全合规**：数据安全、隐私保护、合规审计等安全能力。

### 处理过程

#### 外部视角 - 应用组合关系
从外部视角分析应用组合关系（如场景组合、业务流程组合）：

| 关系类型 | 描述示例 | 具体说明 |
|---------|---------|---------|
| **先后顺序依赖** | 必须先认证才能访问业务 | 前置依赖关系：功能 A 必须先启用或完成才能配置/使用功能 B |
| **前者影响后者** | 用户信息变更影响订单 | 接口变更或配置变更时，承载在本功能上的下游功能会受到影响 |
| **同时发生** | 数据同步和交易同时发生 | 多个操作同时发生时，对系统资源和状态的影响 |
| **共享关系** | 多个服务共享用户中心 | 不同业务场景共享同一核心服务或数据源 |
| **约束关系** | 订单量受库存约束 | 业务指标 A 受到业务指标 B 的约束或限制 |
| **配置互斥** | 两种营销模式无法共存 | 功能 A 和功能 B 互斥，不能同时启用 |

#### 内部视角 - 实现依赖关系
从内部视角分析实现依赖关系（如架构机制耦合关系）：

| 依赖类型 | 描述示例 | 具体说明 |
|---------|---------|---------|
| **缺乏同步过程** | 缓存更新后数据库未同步 | 功能 A 更新后，功能 B 没有同步更新，导致数据不一致 |
| **乱序** | 消息处理顺序错误 | 事件执行顺序错误，影响系统稳定性和业务正确性 |
| **超时** | 接口调用超时 | 操作因超出时间限制而失败，需要超时控制 |
| **消息传递、事务处理交互** | 分布式事务一致性 | 系统间或微服务间需要传递消息或进行事务处理的一致性保证 |
| **共享数据缺陷的相互传递** | 数据错误传播 | 功能特性间共享的数据出现问题或缺陷可能影响多个相关功能 |
| **发生不能共享的资源的访问** | 资源独占性访问 | 资源的访问具有独享、排他性，需要并发控制和锁机制 |
| **死锁** | 循环资源阻塞 | 多个资源互相等待对方释放，导致系统阻塞 |

## 3. 分析方法论

### 外部分析指导原则
1. **场景映射**：将真实世界使用场景映射到功能关系
   - 例如：金融交易的"申请 - 审批 - 放款"业务流程对应系统中的"订单创建 - 风控审批 - 放款执行"功能关系
2. **影响分析**：分析一个组件的变化如何影响其他组件
   - 例如：用户中心修改了用户字段，会影响订单、支付、物流等多个依赖用户信息的系统
3. **资源利用**：识别共享资源及其约束条件
   - 例如：数据库连接池、缓存、消息队列等共享资源的容量和性能约束
4. **配置依赖**：记录配置项之间的前置关系
   - 例如：启用某项高级功能需要先配置基础参数

### 内部分析指导原则
1. **状态同步**：确保依赖组件之间的状态一致性
   - 例如：订单状态在订单服务和支付服务间的一致性
2. **事件排序**：验证关键操作的正确序列
   - 例如：消息队列中消息的顺序处理和事务保证
3. **时间约束**：监控并强制执行时间依赖关系
   - 例如：SLA 要求、超时控制、限流策略
4. **通信模式**：分析系统组件间的消息流
   - 例如：同步调用、异步消息、事件驱动的通信模式
5. **数据完整性**：确保跨组件共享数据的完整性
   - 例如：分布式事务、数据一致性校验
6. **资源竞争**：检测并防止资源访问冲突
   - 例如：数据库锁、缓存锁、分布式锁

### 最佳实践
- 结合外部和内部视角进行全面分析
- 使用行业领域的具体示例记录关系
- 优先关注高优先级关系以聚焦测试重点
- 维护关系矩阵以确保可追溯性
- 建立依赖关系图谱以便可视化管理

## 5. 关系分类标准

### 外部视角关系（业务级）
1. **业务依赖**：业务流程中的前置后置关系
2. **业务约束**：业务规则中的限制条件
3. **业务冲突**：业务规则中的互斥关系
4. **资源共享**：跨业务的资源共享关系

### 内部视角关系（技术级）
1. **状态一致性**：分布式系统中的状态同步
2. **消息传递**：系统间的消息通信
3. **数据共享**：系统间的数据共享和一致性
4. **资源竞争**：并发场景下的资源访问控制
5. **时序依赖**：系统间的时序和超时控制
6. **事务一致性**：分布式事务的数据一致性

## 6. 关系优先级评估

| 优先级 | 评估标准 | 测试策略 |
|--------|----------|----------|
| 高 | 影响核心业务流程、涉及资金安全、数据一致性 | 必须测试，使用自动化测试覆盖 |
| 中 | 影响用户体验、影响系统可用性 | 重点测试，结合自动化和手工测试 |
| 低 | 影响非核心功能、影响有限 | 抽样测试，定期回归验证 |

# 测试模式建模定义与实例（行业解决方案版）

## 测试模式划分

按照 DFX 维度细分为功能、可靠性、性能、安全、兼容性，协议一致性、数据智能、OM、业务交互，提取测试模式。

## 测试模式提取原则

### 可靠性测试模式

【划分原则】
1. 按照可靠性的故障影响层级进行划分，分为业务级、系统级、平台级 3 个层次；
2. 按照故障发生的测试对象、故障类型进行细分，并针对每个故障分析对测试对象可靠性影响；
3. 每层关注本层的保护方案，目标是能在本层解决故障隔离和防扩散。

【编号规则】
`TestOperation"+"-"+"DFR"+"-"+从 001 开始的数字`

【示例】
| 编号 | 测试对象 | 测试操作 | 操作描述 |
|------|----------|----------|----------|
| TestOperation-DFR-001 | 微服务调用 | 服务不可用 | 被调用的微服务不可用，验证调用方的容错处理 |
| TestOperation-DFR-002 | 消息队列 | 消息丢失 | 验证消息持久化和重投机制 |
| TestOperation-DFR-003 | 数据库 | 主节点故障 | 验证主从切换和数据一致性 |
| TestOperation-DFR-004 | 缓存集群 | 节点故障 | 验证缓存节点故障时的自动剔除和恢复 |
| TestOperation-DFR-005 | 支付服务 | 支付超时 | 验证支付超时后的事务处理 |
| TestOperation-DFR-006 | 订单服务 | 订单状态不一致 | 验证订单状态异常后的恢复机制 |

### 性能测试模式

【划分原则】
1. 按照时间和空间维度进行划分，时间性能指业务处理消耗的时间，空间性能指业务处理消耗的内存资源；
2. 时间维度包括响应时间、吞吐量、延迟、收敛时间等；
3. 空间维度包括容量性能、内存占用、资源消耗。

【编号规则】
`TestOperation"+"-"+"DFP"+"-"+从 001 开始的数字`

【示例】
| 编号 | 测试对象 | 测试操作 | 操作描述 |
|------|----------|----------|----------|
| TestOperation-DFP-001 | 交易接口 | 响应时间测试 | 验证交易接口在标准负载下的响应时间指标 |
| TestOperation-DFP-002 | 消息队列 | 吞吐量测试 | 验证消息队列的每秒消息处理量 |
| TestOperation-DFP-003 | 缓存 | 读写性能测试 | 验证缓存的读写性能指标 |
| TestOperation-DFP-004 | 数据库 | 并发查询测试 | 验证数据库在高并发场景下的查询性能 |
| TestOperation-DFP-005 | 风控引擎 | 规则执行性能 | 验证风控规则引擎的执行效率 |
| TestOperation-DFP-006 | 推荐系统 | 推荐生成延迟 | 验证推荐算法生成推荐的耗时 |
| TestOperation-DFP-007 | API 网关 | 并发连接数 | 验证 API 网关的并发连接处理能力 |

### 安全测试模式

【划分原则】
1. 按照 STRIDE 维度进行划分，分别是仿冒、篡改、抵赖、信息泄露、拒绝服务、权限提升；
2. 在此基础上增加隐私保护和合规，其中合规如安全资料、合规报告等；
3. 针对每个维度按照两种方式划分：
   - 基于安全防护逐层突破的思路：如无防护、绕过防护、防护失败等；
   - 基于安全防护内容的聚合：如敏感数据传输、敏感数据处理等。

【编号规则】
`TestOperation"+"-"+"DFS"+"-"+从 001 开始的数字`

【示例】
| 编号 | 测试对象 | 测试操作 | 操作描述 |
|------|----------|----------|----------|
| TestOperation-DFS-001 | 用户账号 | 防仿冒 - 身份认证 | 验证用户身份认证机制的有效性 |
| TestOperation-DFS-002 | 业务数据 | 防篡改 - 数据完整性 | 验证业务数据的完整性校验机制 |
| TestOperation-DFS-003 | 操作日志 | 防抵赖 - 审计追踪 | 验证用户操作的不可抵赖性 |
| TestOperation-DFS-004 | 敏感信息 | 防泄露 - 数据加密 | 验证敏感信息的加密传输和存储 |
| TestOperation-DFS-005 | 服务接口 | 防攻击 - 限流防护 | 验证接口限流和防 DDoS 能力 |
| TestOperation-DFS-006 | 权限控制 | 防提升 - 权限校验 | 验证用户权限校验和越权访问防护 |
| TestOperation-DFS-007 | 个人信息 | 隐私保护 - 数据脱敏 | 验证个人信息的脱敏处理有效性 |
| TestOperation-DFS-008 | 合规日志 | 合规 - 审计记录 | 验证合规审计日志的完整性和可追溯性 |

### 兼容性测试模式

【划分原则】
1. 按照兼容性的类型进行划分：前后向兼容、跨平台兼容、数据兼容、协议兼容；
2. 针对每种类型按照测试对象细分：接口兼容、数据格式兼容、协议版本兼容等。

【编号规则】
`TestOperation"+"-"+"DFC"+"-"+从 001 开始的数字`

【示例】
| 编号 | 测试对象 | 测试操作 | 操作描述 |
|------|----------|----------|----------|
| TestOperation-DFC-001 | API 接口 | 前向兼容 | 验证新版本 API 对旧版本客户端的兼容性 |
| TestOperation-DFC-002 | 数据格式 | 后向兼容 | 验证新版本系统对旧版本数据的兼容性 |
| TestOperation-DFC-003 | 客户端 | 跨平台兼容 | 验证系统在不同操作系统上的兼容性 |
| TestOperation-DFC-004 | 浏览器 | 跨浏览器兼容 | 验证系统在主流浏览器上的兼容性 |
| TestOperation-DFC-005 | 消息协议 | 版本兼容 | 验证消息协议不同版本间的兼容性 |

## 测试操作命名规范

1. **配置类**：以"配置"、"设置"、"创建"、"初始化"等动词开头
2. **验证类**：以"验证"、"检查"、"确认"、"测试"等动词开头
3. **监控类**：以"监控"、"收集"、"查看"、"统计"等动词开头
4. **性能类**：以"测试"、"验证"开头，后接性能指标名称
5. **安全类**：以"验证"、"检查"开头，后接安全维度名称
6. **数据智能类**：以"验证"、"测试"开头，后接数据能力名称

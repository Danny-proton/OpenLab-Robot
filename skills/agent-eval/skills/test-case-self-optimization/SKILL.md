# 测试用例自优化

完成一轮测试后，分析错误分布，迭代增强用例。

## 任务

1. 读上一轮诊断结果
2. 分析 F1-F8 错误分布（哪类失败集中？）
3. 识别 spec 缺口（哪些维度 0 用例？哪些 100% 通过？）
4. 检查用例质量（9 维度：完整性/覆盖度/DFX/二义性/可行性/长度/断言）
5. 生成增强建议
6. 与人确认后更新 cases YAML

## 错误分布→增强策略

| 错误集中 | 增强策略 |
|---------|---------|
| F1-F2 | 补任务类型识别用例 |
| F3 | 增加工具选择边界用例 |
| F4 | 增加参数边界用例 |
| F5 | 增加异常恢复用例 |
| F7 | 细化断言关键词 |
| F8 | 调整 expected_steps |
| 某维度 0 用例 | 补该维度用例 |
| 某维度 100% | 加边界/对抗用例 |

详见 docs/PRD_CASE_SELF_OPTIMIZATION.md。

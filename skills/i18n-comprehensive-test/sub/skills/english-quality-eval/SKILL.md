# 英文语言质量综合评测 Skill

> **背景（Issue 3）**：原 i18n 测试仅检查"英文界面是否有中文残留"与"翻译是否正确"的粗粒度判断，缺乏对英文文本本身的语言质量（语义、语法、表达场合、规范等）的细粒度评测。本子 Skill 提供 7 维度量化评测方法，输出综合评测结果。

## 适用场景

- Phase 5 执行国际化用例时，收集英文界面文本样本
- Phase 1.5 语言包直测时，对英文语言包所有值进行评测
- Phase 5.5 汇总所有英文样本，进行综合评测并输出报告

## 输入

| 输入来源 | 内容 | 何时提供 |
|----------|------|----------|
| UI 文本样本 | Phase 5 各用例收集的英文界面文本（菜单、按钮、表头、提示、占位符等），含截图引用 | Phase 5 完成后 |
| 语言包英文值 | Phase 1.5 解析出的英文语言包所有 key-value | Phase 1.5 完成后（若存在语言包） |
| 上下文信息 | 文本所在页面、UI 元素类型、对应功能 | 与样本一同提供 |

## 评测维度（7 维度）

### 维度 1：词汇准确性（Vocabulary Accuracy）

**检查内容**：
- 选词是否准确表达原意
- 专业术语是否符合行业惯例（如 "Submit" vs "Commit"、"Delete" vs "Remove"）
- 是否存在中式英语（Chinglish）用词
- 动词选择是否恰当（如用 "Save" 还是 "Store"、"Confirm" 还是 "Make Sure"）

**评分标准（0-10）**：
- 9-10：用词精准，专业术语正确
- 7-8：基本准确，个别用词可优化
- 5-6：存在明显不当用词但不影响理解
- 3-4：用词错误，可能引起误解
- 0-2：用词严重错误或完全无法理解

### 维度 2：语法正确性（Grammar Correctness）

**检查内容**：
- 时态使用（现在时/过去时/将来时）是否恰当
- 主谓一致（subject-verb agreement）
- 单复数使用
- 冠词使用（a/an/the）是否正确
- 句法结构是否完整
- 比较级/最高级形式是否正确

**评分标准（0-10）**：
- 9-10：语法完全正确
- 7-8：存在轻微语法瑕疵（如冠词遗漏），不影响理解
- 5-6：有明显语法错误但可推断原意
- 3-4：语法错误导致理解困难
- 0-2：语法严重错误无法理解

### 维度 3：语义表达（Semantic Expression）

**检查内容**：
- 含义是否清晰、无歧义
- 是否完整传达中文原文的语义
- 是否存在过度翻译或漏译
- 语境中是否存在歧义（如 "Bank" 在金融语境 vs 河岸语境）
- 否定/肯定表达是否准确（避免双重否定）

**评分标准（0-10）**：
- 9-10：语义清晰准确，无歧义
- 7-8：基本清晰，个别处可更明确
- 5-6：存在轻微歧义但不影响主要功能
- 3-4：语义模糊，用户可能误判操作后果
- 0-2：语义错误，可能误导用户

### 维度 4：场合规范（Register & Appropriateness）

**检查内容**：
- 正式程度匹配 UI 语境（企业级系统用正式语，社交类可用非正式）
- 用语风格在整页面内一致
- 是否使用了口语化/俚语表达（企业 UI 中通常应避免）
- 礼貌程度（避免生硬命令，但也不能过度客气）
- 按钮文案简洁性（通常 1-3 个词，如 "Save"、"Cancel" 而非 "Please Save It Now"）

**评分标准（0-10）**：
- 9-10：场合完全得体，风格一致
- 7-8：基本得体，个别处风格不一致
- 5-6：存在场合不当（如企业系统用俚语）
- 3-4：风格混乱，影响专业感
- 0-2：严重失当（如错误使用幽默/俚语在严肃场景）

### 维度 5：一致性（Consistency）

**检查内容**：
- 同一术语在不同位置是否一致（如到处用 "Delete" 还是有些地方用 "Remove"）
- 同一操作在不同页面的表述是否一致
- 缩写使用是否一致（如 "No." vs "Number"）
- 大小写规范是否一致（如 Title Case vs Sentence case）
- 与语言包中已有翻译的一致性

**评分标准（0-10）**：
- 9-10：完全一致
- 7-8：个别不一致
- 5-6：存在多处不一致
- 3-4：术语混乱，用户需猜测
- 0-2：完全不一致

### 维度 6：拼写与大小写（Spelling & Capitalization）

**检查内容**：
- 拼写错误（typo）
- Title Case 规范（按钮/标题首字母大写）
- 专有名词大小写（如 "GitHub"、"iPhone"、"JSON"）
- 缩写大小写（如 "ID" vs "Id"、"URL" vs "Url"）
- 句首大写
- 全大写场景的合理性（如警告/强调）

**评分标准（0-10）**：
- 9-10：拼写与大小写完全正确
- 7-8：个别大小写不规范
- 5-6：存在拼写错误或大小写不一致
- 3-4：多处拼写错误
- 0-2：拼写错误严重，影响可读性

### 维度 7：标点符号（Punctuation）

**检查内容**：
- 中英文标点混用（如英文文案用了中文逗号"，"或句号"。"）
- 标点使用规范（Oxford comma、句末标点）
- 省略号、破折号使用是否正确
- 引号使用（直引号 vs 弯引号）
- 是否多余或缺失标点
- 冒号/分号使用是否恰当

**评分标准（0-10）**：
- 9-10：标点完全规范
- 7-8：个别标点不规范
- 5-6：存在中英文标点混用
- 3-4：多处标点错误
- 0-2：标点混乱，影响阅读

## 评测流程

### Step 1：样本归集

汇总所有英文文本样本，每条样本结构化记录：

```
样本ID: <S-001>
来源: <UI|语言包>
页面: <页面名 / 语言包文件名>
元素类型: <菜单|按钮|表头|提示|占位符|标签|语言包key>
英文文本: "<实际英文文本>"
中文原文: "<对应中文，若已知>"
截图引用: <截图文件路径，UI 样本必填>
上下文: <该文本所在的功能/操作场景>
```

### Step 2：逐样本 7 维度评分

对每条样本，按 7 维度独立评分（0-10），并记录问题：

```
样本: S-001
英文文本: "Please input you username"
维度1 词汇准确性: 8/10  - "input" 可优化为 "enter"
维度2 语法正确性: 3/10  - "you" 应为 "your"（主谓所有格错误）
维度3 语义表达:   7/10  - 语义可理解但有语法瑕疵
维度4 场合规范:   8/10  - "Please input" 略冗长，"Enter username" 更佳
维度5 一致性:     7/10  - 需对照其他位置同类提示
维度6 拼写大小写: 6/10  - "you" 拼写错误（应为 your）；句首大写正确
维度7 标点符号:  10/10  - 无标点问题
样本均分: 5.9/10
问题: "you" 应为 "your"，属语法与拼写双重错误
建议: 改为 "Enter your username" 或 "Please enter your username"
```

### Step 3：维度级汇总

汇总所有样本的各维度均分：

```
维度1 词汇准确性:    全样本均分 X.X/10
维度2 语法正确性:    全样本均分 X.X/10
维度3 语义表达:      全样本均分 X.X/10
维度4 场合规范:      全样本均分 X.X/10
维度5 一致性:        全样本均分 X.X/10
维度6 拼写与大小写:  全样本均分 X.X/10
维度7 标点符号:      全样本均分 X.X/10
综合均分:           X.X/10
```

### Step 4：综合质量等级

按综合均分映射质量等级：

| 综合均分 | 等级 | 含义 |
|----------|------|------|
| 9.0-10 | A | 优秀，可直接发布 |
| 7.0-8.9 | B | 良好，少量优化项 |
| 5.0-6.9 | C | 合格，存在明显问题需修复 |
| < 5.0 | D | 不合格，需大规模返工 |

### Step 5：问题清单与修复建议

按维度归类问题，每条问题记录：

```
问题ID: Q-001
维度: 语法正确性
严重度: <Critical | Major | Minor>
样本: [S-001, S-005, S-012]
问题描述: 多处 "you" 误用为所有格应为 "your"
修复建议: 全局替换 "you username/you password" → "your username/your password"
影响范围: 登录页、注册页、找回密码页
```

### Step 6：输出综合评测报告

保存为 `ENGLISH_QUALITY_EVAL_REPORT.md`，结构见下方模板。

## 评测原则

1. **客观量化**：每个维度独立评分，避免主观笼统判断
2. **结合上下文**：评分时考虑文本所在的 UI 元素类型与功能场景
3. **交叉验证**：UI 样本与语言包样本（若存在）交叉验证一致性
4. **问题可追溯**：每个问题关联具体样本与截图，便于定位修复
5. **建议可执行**：修复建议具体到操作（如"将 X 改为 Y"），避免空泛
6. **不替代残留检查**：本评测聚焦英文文本质量，"中文残留于英文界面"由 Checklist 项 1 负责，两者互补

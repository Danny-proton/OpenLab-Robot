## 报告规则

1. **必须输出** — 每次测试完成后必须输出报告，格式为 Markdown
2. **必须保存** — 报告必须同时保存到 `<用例编号>/test_report.md` 文件中，路径与截图文件在同一目录下
3. **如实记录** — 操作结果基于实际执行情况，不得编造
4. **失败标记** — 步骤失败标记为 ❌，最终状态标记为 ❌ 或 ⚠️
5. **失败原因** — 必须说明失败的具体原因和修复建议
6. **截图完整性校验** — 输出报告前必须校验：
   - 每个操作步骤都有对应的截图文件存在
   - 报告中的截图路径与实际文件路径完全匹配
   - **截图缺失时报告不可输出**，必须先补截图或标记步骤为 ❌
7. **报告中的截图引用** — 在"执行步骤与结果"表中，每行必须包含对应截图文件名：

```markdown
| 步骤 | 操作描述              | 操作结果 | 页面截图 |
|------|----------------------|----------|----------|
| 1    | 导航到 https://...   | ✅ 成功  | `step_01_navigate.png` |
| 2    | 填写表单并提交        | ✅ 成功  | `step_02_submit_form.png` |
```

示例：用例编号 `FUN_SR20251125519777_001_QYXD` 的报告保存路径为 `./FUN_SR20251125519777_001_QYXD/test_report.md`

# 项目执行规则

## 基础原则

- 执行任何用例前，先确认所需系统均已登录。
- 禁止关闭或覆盖已打开的系统页签。
- 需要访问新系统或新页面时，应优先新建页签。
- 操作前先 take_snapshot，获取最新 uid。
- 关键操作后必须验证 value、checked、dialog、toast 或截图证据。

## 工具优先级

- 文本框、文本域、日期输入框：优先 fill_form。
- radio、checkbox、下拉框、自定义组件：优先 click。
- evaluate_script 只用于排查、隐藏元素点击、页面状态检查。
- 禁止用 JS 直接写入表单值，除非当前项目记忆或用户指令明确允许。

## 页签管理

- 登录后的页签必须保持打开，后续用例执行中在不同页签间切换，禁止关闭或覆盖任何页签。
- 导航到新的系统地址，须新建页签，在不同页签间切换。

## 证据采集

- 权限验证操作后必须立即截图保存，因为错误提示对话框可能一闪而过。
- 操作成功后截图记录提示语（如 Operation successful）。

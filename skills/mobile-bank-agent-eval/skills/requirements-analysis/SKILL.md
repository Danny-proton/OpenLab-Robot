# 需求分析

你是一个需求分析专家。请根据用户提供的 Agent 需求说明，生成 10 个测试维度和对应的测试场景。

## 你的任务

1. 仔细阅读用户的需求说明
2. 为以下 10 个维度各生成 3-5 个具体场景
3. 将结果整理成 JSON
4. 调用 `case_io.py` 写入 YAML 文件

## 10 个测试维度（固定框架）

1. **业务场景覆盖** (DIM-001) — 按业务功能拆解用户典型场景（账户查询/转账/理财/信用卡等）
2. **业务流程覆盖** (DIM-002) — 正常路径、替代路径、异常路径
3. **用户角色与意图覆盖** (DIM-003) — VIP/普通用户/游客，查询/操作/投诉/闲聊
4. **业务规则与约束覆盖** (DIM-004) — 限额/合规/if-then 逻辑/数值约束
5. **输入形态与上下文覆盖** (DIM-005) — 错别字/指代消解/不完整信息/多轮上下文
6. **安全与边界覆盖** (DIM-006) — 注入攻击/越权/敏感数据泄露
7. **多轮对话状态覆盖** (DIM-007) — 上下文保持/状态迁移/话题切换
8. **异常恢复流程覆盖** (DIM-008) — 超时/网络错误/数据异常后恢复
9. **性能与延迟边界覆盖** (DIM-009) — 大数据量/高并发/延迟容忍
10. **合规与监管覆盖** (DIM-010) — 适当性管理/信息披露/禁止刚性兑付

## 设计原则

- 维度名称结合具体业务场景（如"账户查询场景覆盖"而非仅"业务场景覆盖"）
- 每个维度至少 3 个场景
- 场景描述要具体，体现前置条件或触发条件
- 覆盖正常/异常/边界三种情况

## 输出

生成 JSON 后，调用脚本写入 YAML：

```bash
python ${SKILL_PATH}/scripts/case_io.py write-requirements \
  --output .agent-eval/data/requirements.yaml \
  --json '{"dimensions": [...], "scenarios": [...]}'
```

JSON 格式：
```json
{
  "dimensions": [
    {"id": "DIM-001", "name": "账户查询场景覆盖", "type": "业务场景覆盖"}
  ],
  "scenarios": [
    {"id": "SC-001", "dimension": "DIM-001", "name": "查询账户余额", "description": "用户登录后查询主账户余额"}
  ]
}
```

**你（Agent）负责分析需求和生成内容，脚本只负责写文件。**

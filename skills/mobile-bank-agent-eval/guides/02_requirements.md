# Guide 02 — 需求分析

10 个测试维度，覆盖手机银行 Agent 全场景。

## 维度列表

| ID | 维度 | 覆盖类型 | 示例场景 |
|----|------|---------|---------|
| DIM-001 | 业务场景覆盖 | 场景 | 账户查询/转账/理财 |
| DIM-002 | 业务流程覆盖 | 流程 | 正常/替代/异常路径 |
| DIM-003 | 用户角色与意图覆盖 | 角色 | VIP/普通用户 |
| DIM-004 | 业务规则与约束覆盖 | 规则 | 限额/合规 |
| DIM-005 | 输入形态与上下文覆盖 | 输入 | 错别字/指代 |
| DIM-006 | 安全与边界覆盖 | 安全 | 注入/越权 |
| DIM-007 | 多轮对话状态覆盖 | 状态 | 上下文保持 |
| DIM-008 | 异常恢复流程覆盖 | 异常 | 超时恢复 |
| DIM-009 | 性能与延迟边界覆盖 | 性能 | 大数据量 |
| DIM-010 | 合规与监管覆盖 | 合规 | 适当性管理 |

## 输出

- Excel: 3 个 sheet（测试维度/测试场景/Skill归属建议）
- Gherkin: .feature 文件（可选，--gherkin）

## LLM

- 有 LLM_API_KEY: 调真实 LLM 生成
- 无 LLM_API_KEY: 用 mock LLM 生成预定义维度

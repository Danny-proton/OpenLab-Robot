# 设计系统：华为云上 OpenLab

> 本文档总结 https://openlab.huawei.com/partner/#/home 的视觉设计语言，作为本仓库（OpenLab-Robot）项目前端开发的设计基线。
>
> 采集来源：云上 OpenLab 公开落地页（`/partner/home/`）、Uniportal 统一登录页样式，以及华为品牌设计惯例。内层工作台（`#/home`）需 Uniportal 账号登录后方可访问，其规范以本文件推导的通用规则为准。格式参考 awesome-design-md-cn 的 DESIGN.md 模板。

## 1. 视觉主题与氛围

云上 OpenLab 的设计语言是「企业级科技感 + 华为品牌红点缀」。整体氛围干净、明亮、专业：页面以浅蓝灰（`#EBEFF5`）为底色，顶部铺一张浅蓝渐变（`#D8E3F8 → #E4EDFE → #E8F0FD`）的科技插画背景图（3D 蓝色云朵、轨道、芯片等元素），内容则承载在纯白（`#FFFFFF`）大圆角卡片上，形成「浅色科技背景 + 白色浮起卡片」的经典企业门户结构。

色彩策略极为克制：全站唯一的强品牌色是华为红（`#C7000B`），仅用于 Logo、主标题中的强调字（如「云上 **O**penLab」的红色 O）和悬停高亮；链接与次级强调使用沉稳的企业蓝（`#305FBF` / `#0067D1`）。文字不是纯黑，而是偏蓝灰的近黑（`#24282E` / `#474E57`），与浅蓝背景在色调上保持同一冷色家族，整体观感统一、冷静、可信赖。

**关键特征：**

- 华为红 `#C7000B` 作为唯一品牌强调色，用量极少、只点关键位置
- 浅蓝灰页面底色 `#EBEFF5` + 浅蓝渐变科技插画头图，营造「云上」氛围
- 白色大圆角（16px）内容卡片，悬停时浮起柔和阴影，是可点击的视觉隐喻
- 近黑蓝灰文字（`#24282E` 标题 / `#474E57` 正文），不用纯黑
- 1px 浅灰竖向分割线（`#DCE0E6`）在卡片组内做轻量区隔
- 中英双语并行，中文优先，英文作为副标题小字呈现
- 插画风格：3D 等距科技插画（蓝白主调）+ 细线条线性图标，不用照片

## 2. 色彩体系与角色

### 品牌色

- **华为红 Huawei Red**（`#C7000B`）：品牌标识、标题强调字、文字/图标悬停高亮、登录主按钮与关键链接（Uniportal 红色主题）。全站唯一饱和暖色，克制使用。
- **企业蓝 Link Blue**（`#305FBF`）：落地页下划线链接的悬停色。
- **国际蓝 Intl Blue**（`#0067D1`）：Uniportal 国际站主题色，复选框选中态、蓝色文字链接。

### 中性色阶（蓝灰家族）

- **近黑 Title Black**（`#24282E` / `rgba(36,40,46,1)`）：卡片标题、移动端标题、大标题文字。
- **深灰 Header Text**（`#333333`）：顶部导航文字。
- **正文灰 Body Gray**（`#474E57` / `rgba(71,78,87,1)`）：描述文字、副标题、卡片说明——全站使用量最大的文字色。
- **弱灰 Muted Gray**（`#828994`）：页脚文字、法律声明、备案信息。
- **页脚深灰**（`#282828`）：页脚链接区标题与正文（`#F0F0F0` 底上）。

### 背景与表面

- **页面底色 Page BG**（`#EBEFF5`）：全局背景、页脚背景。
- **卡片白 Surface White**（`#FFFFFF`）：所有内容卡片、次级卡片的表面色。
- **移动卡片灰**（`#F5F7FA` / `rgba(245,247,250,1)`）：移动端插图槽位背景。
- **页脚分区灰**（`#F0F0F0`）：页脚链接区背景。
- **Hero 渐变**（`#D8E3F8 → #E4EDFE → #E8F0FD`）：顶部科技插画背景的浅蓝渐变区间。

### 线条与分割

- **分割灰 Divider**（`#DCE0E6`）：卡片组内 1px 竖向分割线。
- **页脚顶边**（`rgba(255,255,255,0.09)`）：页脚 1px 顶部边线。

### 语义色（继承 Element UI / Vant 体系，用于表单与反馈）

- **成功绿**（`#67C23A`）：成功状态。
- **警告橙**（`#E6A23C`）：警告状态。
- **错误红**（`#F56C6C` / Vant `#EE0A24`）：错误与危险操作。
- **信息灰**（`#909399`）：辅助信息。

### 阴影

- **卡片悬停阴影**（`0 2px 20px 0 rgba(0,0,0,0.10)`）：唯一的阴影规格，用于卡片 hover 浮起；默认态卡片无阴影，靠白卡与灰底的明度差分层。

## 3. 字体排版规则

### 字体族

- **中文主字体**：`PingFang SC`，回退 `Helvetica Neue, Arial, sans-serif`（Windows 端回退 `Microsoft YaHei`）。
- **英文/数字强调**：`Avenir-Heavy` 用于需要重量感的英文标题场景。
- 全站无衬线，不用等宽字体做装饰。

### 层级

| 角色 | 字号 | 字重 | 行高 | 颜色 | 说明 |
|------|------|------|------|------|------|
| Hero 主标题 | 约 64-80px（图形化 Logo 字） | 700 | 1.0 | `#24282E` + 红强调字 | 「云上 OpenLab」，红 O 点睛 |
| Hero 描述 | 17px | 400 | 1.5 | `#474E57` | 一句话平台定位说明 |
| 一级卡片标题 | 25px | 400-600 | 1.2 | `#24282E` | 四条流水线名称 |
| 英文副标题 | 约为中文的 62% | 400 | 1.5 | `#474E57` | 卡片内英文对照 |
| 二级卡片标题 | 19px | 400 | 1.2 | `#24282E` | 技术支持 / 帮助文档 |
| 卡片描述 | 15px | 400 | 1.5 | `#474E57` | 功能简述 |
| 导航文字 | 16px | 400 | 36px（行内居中） | `#333333` | 语言切换等，hover 变红 |
| 页脚链接 | 14px | 400 | 22px | `#828994` | 底部导航链接 |
| 页脚法律信息 | 12px | 400 | 18px | `#828994` | 版权、备案号 |
| 表单正文（登录） | 14px | 400 | 1.5 | `#323233` | 输入框、表单标签 |
| 表单辅助 | 12px | 400 | 1.5 | `#969799` | 占位符、帮助文本 |

### 原则

- **字号克制**：全站常用字号集中在 12 / 14 / 15 / 16 / 17 / 19 / 25px，没有花哨的超大号梯队；层级靠颜色深浅和位置而非字号差拉开。
- **行高统一 1.5**：所有描述性文字行高均为 1.5，保证企业文档式的阅读密度。
- **中英混排**：中文为主标题，英文缩小至约 62% 作为对照副标题，颜色降为 `#474E57`。
- **标题不纯黑**：标题用 `#24282E`，比纯黑柔和，与蓝灰中性色同族。

## 4. 组件样式

### 按钮

**主按钮（登录/关键操作，Uniportal 红色主题）**

- 背景：`#C7000B`
- 文字：`#FFFFFF`，14px
- 圆角：4px
- 悬停：明度微降，无位移
- 用途：登录、提交等唯一主操作

**文字链接按钮**

- 默认：`#474E57`，带下划线（落地页 POC 赋能 / 实操演练 / 在线考试等入口）
- 悬停：`#305FBF`（蓝）或 `#C7000B`（导航/页脚场景，红）
- 间距：并列链接间隔 24px

**语言切换**

- 形态：纯文字（如 `English`），无按钮外壳
- 默认 `#333333`，悬停 `#C7000B`

### 卡片与容器

**一级功能卡片（四条流水线）**

- 尺寸：393.75 × 432px，四列横排
- 背景：`#FFFFFF`，圆角 16px
- 内边距：3%
- 默认：无阴影、无边框
- 悬停：`box-shadow: 0 2px 20px 0 rgba(0,0,0,0.10)`，整卡可点击（`cursor: pointer`）
- 结构：标题（25px）→ 英文副标题 → 线性插画（200px 高）→ 描述（15px）
- 卡间以 1 × 240px 的 `#DCE0E6` 竖线分隔

**二级功能卡片（技术支持 / 帮助文档）**

- 尺寸：780 × 227px，两列
- 背景：`#FFFFFF`，圆角 16px，同样的悬停阴影
- 结构：细线图标（40px）→ 标题（19px）→ 描述或下划线链接组（15px）

**移动端卡片**

- 全宽白卡，圆角 16px，内边距 1.6rem，纵向堆叠
- 插图槽位：`#F5F7FA` 底、12px 圆角

### 输入与表单（Uniportal）

- 输入框：白底，1px 边框（`#C0C4CC` 系），圆角 4px，高度约 40px
- 聚焦：边框转主题色（红主题 `#C7000B` / 国际蓝 `#0067D1`）
- 复选框选中态：`#0067D1`
- 占位符：`#969799`，12-14px

### 导航

- 顶栏高 72px，左右内边距 40px，绝对定位于 Hero 之上，默认透明
- 左侧：华为 Logo（红色花瓣 + 黑色 HUAWEI 字标，138 × 36px）
- 右侧：语言切换文字
- 无传统多链接菜单——落地页是单入口集散页，导航极简

### 图像处理

- Hero 插画：3D 等距科技风，蓝白主调（云朵、轨道、芯片、摄像头），约占首屏右半
- 卡片插图：细线条线性插画（黑线稿 + 少量蓝色点缀），200px 高，居中
- 图标：细线风格（耳机、书本），40px 上下，不用实心面性图标
- Logo 与插画使用 SVG，高分屏（>3000px）切换 2x 资源

## 5. 布局原则

### 间距体系

- 基础单位：8px；常用步进 8 / 16 / 24 / 32 / 40px
- 卡片内边距：3%（大卡）、16px（次级卡与移动端）
- 并列文字链接间距：24px
- 页脚内边距：32px 0

### 网格与容器

- 内容居中，`width: fit-content; margin: auto` 的居中策略，无刚性 max-width 容器
- 首屏结构：72px 顶栏 → Hero（左文右图，左栏约 645px 宽、缩进 7.4%）→ 四列白卡组 → 两列次级卡组 → 页脚
- 页脚链接区：`margin: auto 260px`（小屏收敛到 100px / 25px），五列均分（每列 20%）

### 留白哲学

- **背景即留白**：页面底色 `#EBEFF5` 直接作为分区手段，卡片组之间靠底色间隔，不加额外分割线
- **垂直节奏**：`padding-top: 6%`（body）、卡片区距 Hero 约 7.5%，用百分比留白适配高度
- **卡片内呼吸感**：卡片内容居中排布，上下留白大于左右，插画与文字之间留 10-16px

### 圆角阶梯

- 4px：按钮、输入框等表单控件
- 12px：移动端插图槽位
- 16px：所有内容卡片（桌面 + 移动统一）
- 100% / 50%：头像、圆形图标容器

## 6. 深度与层级

| 层级 | 处理 | 用途 |
|------|------|------|
| Level 0 平面 | 无阴影无边框，`#EBEFF5` 底 | 页面背景、页脚 |
| Level 1 表面 | 白卡 `#FFFFFF`，16px 圆角，无阴影 | 所有内容卡片的默认态——靠白/灰明度差浮于底色之上 |
| Level 2 浮起 | `0 2px 20px 0 rgba(0,0,0,0.10)` | 卡片悬停态，唯一的阴影规格 |
| Level 3 分割 | 1px `#DCE0E6` 竖线 | 卡片组内部的轻量区隔 |

**阴影哲学**：全站只有一档阴影，且只在悬停时出现。默认态依靠「白卡片 + 灰蓝底」的明度对比建立层级，交互时才用柔和弥散阴影（20px 模糊、0.10 透明度）给出浮起反馈——克制、轻量、企业级。禁止生硬的多层投影和浓黑阴影。

### 装饰性深度

- Hero 区靠背景插画的 3D 元素制造纵深感，UI 元素本身保持扁平
- 分区完全依赖背景色与留白，不使用粗重分隔线

## 7. 响应式行为

### 断点

| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| 超大屏 | ≥3000px | Logo/插画切换 2x 资源，卡片放大到 690 × 370px |
| 大桌面 | 1537-1900px | 全部卡片尺寸 × 0.91 |
| 标准桌面 | 1281-1536px | 卡片 × 0.8，卡片标题 22px |
| 小桌面 | 961-1280px | 卡片 × 2/3，页脚边距收敛到 100px |
| 平板 | 768-960px | 卡片 × 0.5，标题 14px，卡片描述 scale(0.7) |
| 手机 | <768px | 全面切换 rem 布局：卡片纵向堆叠、插图槽左图右文、竖分割线隐藏、背景换移动版插画 |

### 收缩策略

- 桌面端（≥768px）：不按栅格重排，而是整套卡片按比例等比缩放（0.91 / 0.8 / 2/3 / 0.5 四档），保持四列构图
- 移动端（<768px）：彻底换布局——四列卡片变为纵向堆叠的全宽卡，插图从「上图下文」变为「左图右文」，字号改用 rem（标题 1.6rem / 正文 1.2rem）
- 顶栏：72px → 3.4rem 高，Logo 等比缩小
- 页脚：五列 → 纵向居中堆叠，文字降为 1rem

### 触控目标

- 整卡即点击区域（393 × 432px），移动端卡片全宽
- 文字链接行高 ≥ 22px，表单控件高 ≥ 40px

## 8. 可访问性与状态

### 对比度

- 标题 `#24282E` 于白底：约 15:1（WCAG AAA）
- 正文 `#474E57` 于白底：约 8.5:1（WCAG AAA）
- 页脚 `#828994` 于 `#EBEFF5`：约 3.5:1（仅用于次要法律文本）
- 华为红 `#C7000B` 于白底：约 6.5:1（WCAG AA，可用于正文级链接）

### 交互状态

- **默认**：白卡无阴影，链接 `#474E57` 带下划线
- **悬停**：卡片浮起阴影；文字链接变 `#305FBF`（内容区）或 `#C7000B`（导航/页脚）；悬停即 `cursor: pointer`
- **聚焦**：表单控件边框转主题色（红/蓝）；全站应保证键盘可达
- **禁用**：文字降灰（`#969799` / `#C0C4CC` 系），降低对比

### 示例组件提示词

- 「做一个落地页 Hero：页面底色 `#EBEFF5`，顶部铺浅蓝渐变（`#D8E3F8 → #E8F0FD`）科技插画风背景。左侧大标题 64px+、字重 700、颜色 `#24282E`，其中一个关键字母用华为红 `#C7000B` 点睛；下方描述 17px、行高 1.5、颜色 `#474E57`、宽度约 645px。右侧放蓝白主调的 3D 等距科技插画。顶栏 72px 高、透明底、左右内边距 40px，左 Logo 右语言切换（16px `#333333`，hover 变 `#C7000B`）。」
- 「做一个四列功能卡组：每张卡 393.75 × 432px、白底、16px 圆角、无默认阴影、3% 内边距、整卡可点击；悬停加 `0 2px 20px 0 rgba(0,0,0,0.10)` 阴影。卡内自上而下：25px 标题（`#24282E`）、英文副标题（`#474E57`，约 15px）、200px 高细线条线性插画、15px 描述（`#474E57`，行高 1.5）。卡与卡之间用 1 × 240px 的 `#DCE0E6` 竖线分隔。」
- 「做一个次级双列卡：780 × 227px、白底 16px 圆角、同样的悬停阴影；内容为 40px 细线图标 + 19px 标题 + 15px 描述，或一组带下划线的文字链接（默认 `#474E57`，hover `#305FBF`，间距 24px）。」
- 「做一个主按钮：背景 `#C7000B`、白字 14px、4px 圆角、高度约 40px，悬停仅做明度微降；表单输入框白底、1px `#C0C4CC` 边框、4px 圆角，聚焦时边框转 `#C7000B`。」
- 「做页脚：背景与页面同色 `#EBEFF5`，链接区五列均分，链接 14px `#828994`、行高 22px，hover 加下划线或变 `#C7000B`；底部法律信息 12px，链接间用 1px `#828994` 竖线分隔。」

### 迭代守则

1. 华为红 `#C7000B` 是唯一暖色，只点关键位置（Logo、标题强调字、主按钮、悬停高亮），绝不大面积铺色
2. 中性色全部走蓝灰家族（`#24282E` / `#474E57` / `#828994` / `#EBEFF5`），禁止暖灰、米色混入
3. 卡片默认无阴影，层级靠「白卡 + 灰蓝底」明度差；阴影只有一档 `0 2px 20px 0 rgba(0,0,0,0.10)` 且只在 hover 出现
4. 圆角只有三档：4px（表单）、12px（移动插图槽）、16px（卡片）
5. 描述文字行高恒为 1.5；中文为主、英文缩至约 62% 做副标题
6. 图标用细线条线性风格，插画用蓝白 3D 等距科技风，不用照片和实心面性图标
7. 桌面端卡片组按 0.91 / 0.8 / 2/3 / 0.5 四档等比缩放，<768px 才切换纵向堆叠移动布局
8. 分区靠底色与留白，不用粗分割线；页脚与页面同底色，不做深色页脚
